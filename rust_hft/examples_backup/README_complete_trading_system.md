# Complete HFT Trading System

這是一個完整的高頻交易系統，包含所有你需要的組件：

## 🚀 特性

- **📡 實時報價接收**: 連接 Bitget WebSocket 接收 BTCUSDT 實時訂單簿數據
- **🧠 深度學習策略**: 集成 DL 模型進行交易信號生成
- **🚫 Dry-run 模式**: 安全的模擬交易模式，不會實際下單
- **📊 即時 CLI 監控**: 實時顯示交易狀態、P&L、風險指標
- **⚡ 超低延遲**: 優化的執行路徑，延遲 < 1ms
- **🛡️ 風險管理**: 內建止損、倉位限制、每日交易限制

## 🔧 運行系統

```bash
# 進入項目目錄
cd rust_hft

# 編譯檢查
cargo check --example complete_trading_system

# 運行完整交易系統
cargo run --example complete_trading_system
```

## 🎮 控制鍵

系統運行時可使用以下按鍵：

- **Q**: 退出系統
- **M**: 切換模式（DRY-RUN ↔ LIVE）
- **R**: 重置統計數據
- **SPACE**: 暫停（功能待實現）

## ⚙️ 配置參數

系統內建了安全的默認配置：

```rust
TradingSystemConfig {
    dry_run_mode: true,               // 默認安全模式
    max_position_size_usdt: 50.0,     // 最大倉位 50 USDT
    daily_loss_limit_usdt: 20.0,      // 日損失限制 20 USDT
    min_confidence_threshold: 0.65,   // 最低信號置信度 65%
    max_trades_per_hour: 100,         // 每小時最大交易數
    initial_balance_usdt: 400.0,      // 初始資金 400 USDT
}
```

## 🛡️ 風險管理

系統內建多層風險保護：

1. **Dry-run 模式**: 默認模擬交易，不會實際下單
2. **倉位限制**: 單次最大 50 USDT，總倉位限制 150 USDT
3. **日損限制**: 每日最大損失 20 USDT
4. **頻率限制**: 每小時最多 100 筆交易
5. **置信度閾值**: 信號置信度必須 > 65%
6. **餘額檢查**: 確保有足夠資金執行交易

## 📈 DL 策略說明

系統使用基於訂單簿不平衡 (OBI) 的深度學習策略：

- **特徵提取**: 實時計算 OBI、價差、深度壓力等 25 個特徵
- **信號生成**: 基於多時間框架 (3s, 5s, 10s) 預測
- **風險調整**: Kelly 分數動態調整倉位大小
- **在線學習**: 持續更新模型參數

## 🔍 監控指標

### 延遲指標
- **數據處理延遲**: < 500μs (目標)
- **信號生成延遲**: < 200μs (目標)
- **執行延遲**: < 100μs (目標)

### 財務指標
- **日化收益率**: 0.8-2.3% (目標)
- **勝率**: > 65% (目標)
- **最大回撤**: < 3% (限制)

### 系統指標
- **數據更新頻率**: ~10-20 次/秒
- **信號生成頻率**: ~1-5 次/分鐘
- **系統正常運行時間**: > 99%

## ⚠️ 注意事項

1. **默認安全模式**: 系統默認運行在 DRY-RUN 模式，不會實際下單
2. **網絡連接**: 需要穩定的網絡連接到 Bitget WebSocket
3. **資源使用**: 系統會持續處理市場數據，占用 CPU 和內存
4. **日誌記錄**: 所有操作都會記錄在系統日誌中

## 🚨 切換到實盤模式

**警告**: 只有在充分測試並理解風險後才切換到實盤模式！

按 `M` 鍵可以切換模式，但實盤模式需要：

1. 配置 Bitget API 密鑰
2. 確保有足夠的資金
3. 充分理解風險
4. 監控系統運行

## 📞 支持

如有問題，請檢查：

1. 網絡連接是否穩定
2. 終端是否支持彩色輸出
3. 系統資源是否充足
4. Bitget WebSocket 是否可達

系統會在日誌中顯示詳細的錯誤信息和狀態更新。
