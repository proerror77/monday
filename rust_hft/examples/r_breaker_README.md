# R-Breaker Live Trading System

R-Breaker 是一個經典的日內反轉交易策略，本實現提供了完整的實時交易系統，支持 1 分鐘 K 線數據的 R-Breaker 策略。

## 策略原理

R-Breaker 策略基於前一交易日的高、低、收盤價計算出六個關鍵價位：

1. **突破買入價 (Breakthrough Buy)** = H + 2×f×(P - L)
2. **觀察賣出價 (Observation Sell)** = P + f×(H - L)  
3. **反轉賣出價 (Reversal Sell)** = 2×P - L
4. **反轉買入價 (Reversal Buy)** = 2×P - H
5. **觀察買入價 (Observation Buy)** = P - f×(H - L)
6. **突破賣出價 (Breakthrough Sell)** = L - 2×f×(H - P)

其中：
- H = 前日最高價
- L = 前日最低價  
- C = 前日收盤價
- P = (H + L + C) / 3 (樞軸點)
- f = 敏感度係數 (默認 0.35)

## 交易邏輯

### 趨勢跟蹤
- 當價格突破 **突破買入價** 時，執行多頭交易
- 當價格跌破 **突破賣出價** 時，執行空頭交易

### 反轉交易  
- 當價格曾觸及 **觀察賣出價** 後回落至 **反轉賣出價** 時，執行空頭交易
- 當價格曾觸及 **觀察買入價** 後反彈至 **反轉買入價** 時，執行多頭交易

## 使用方法

### 基本運行

```bash
# 編譯項目
cargo build --release

# 運行 R-Breaker 實時交易（模擬模式）
cargo run --example r_breaker_live_trading -- --simulation

# 運行指定交易對和參數
cargo run --example r_breaker_live_trading -- \
    --symbol ETHUSDT \
    --capital 50000 \
    --sensitivity 0.4 \
    --duration 120 \
    --verbose
```

### 命令行參數

| 參數 | 類型 | 默認值 | 說明 |
|------|------|--------|------|
| `--symbol` | String | BTCUSDT | 交易對符號 |
| `--capital` | f64 | 10000.0 | 初始資金 |
| `--max-position-ratio` | f64 | 0.1 | 單筆最大倉位比例 |
| `--stop-loss` | f64 | 0.02 | 止損比例 (2%) |
| `--take-profit` | f64 | 0.04 | 止盈比例 (4%) |
| `--sensitivity` | f64 | 0.35 | R-Breaker 敏感度係數 |
| `--duration` | u64 | 60 | 運行時間（分鐘） |
| `--verbose` | flag | false | 詳細輸出模式 |
| `--simulation` | flag | false | 模擬交易模式 |

### 配置文件

使用配置文件進行更詳細的設置：

```bash
# 使用自定義配置文件
cargo run --example r_breaker_live_trading -- \
    --config config/r_breaker_config.yaml
```

## 參數調優建議

### 敏感度係數 (sensitivity)
- **0.2-0.3**: 較保守，減少假信號但可能錯過機會
- **0.35**: 標準設置，平衡信號數量和質量  
- **0.4-0.5**: 較激進，增加交易頻率但可能增加噪音

### 止損止盈設置
- **保守型**: 止損 1.5%, 止盈 3%
- **標準型**: 止損 2%, 止盈 4%
- **激進型**: 止損 2.5%, 止盈 5%

### 倉位管理
- **保守型**: 單筆倉位 5-8%
- **標準型**: 單筆倉位 10-15%
- **激進型**: 單筆倉位 15-20%

## 注意事項

1. **模擬交易**: 首次使用建議開啟 `--simulation` 模式進行測試
2. **市場條件**: R-Breaker 在震盪市場中表現較好，趨勢市場中需要謹慎
3. **參數調整**: 不同交易對可能需要調整敏感度係數
4. **風險控制**: 建議設置合理的止損和每日虧損上限
5. **網絡穩定**: 確保網絡連接穩定，避免數據中斷影響交易

## 故障排除

### 常見問題

1. **WebSocket 連接失敗**
   ```
   ERROR: WebSocket connection failed
   解決方案: 檢查網絡連接，確認 Bitget API 可用性
   ```

2. **K 線數據不足**
   ```
   WARN: Insufficient kline data for R-Breaker calculation
   解決方案: 等待更多數據累積，或使用歷史數據初始化
   ```

3. **交易信號過於頻繁**
   ```
   解決方案: 增加敏感度係數或添加信號過濾條件
   ```
