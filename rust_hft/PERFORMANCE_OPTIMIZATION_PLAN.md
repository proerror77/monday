# 性能優化方案

## 🔍 **當前問題分析**

### 1. **吞吐量低下**
- 當前：8.5 msg/s
- 期望：100+ msg/s
- 差距：92% 的性能缺失

### 2. **連接失活**
- 5分鐘後停止接收數據
- ping/pong 機制失效
- 連接被錯誤關閉

### 3. **負載均衡問題**
- 均衡比例：0.53 < 0.80
- 自動重新平衡未生效

## 🎯 **立即優化方案**

### 1. **增強 ping/pong 機制**
```rust
// 問題：過於敏感的 pong 檢查
if !pong_received {
    break; // 立即斷開
}

// 優化：容忍 3 次連續無響應
if CONSECUTIVE_MISSING_PONGS >= 3 {
    break;
}
```

### 2. **延長連接超時**
```rust
// 問題：2分鐘無消息就斷開
if last_message_time.elapsed() > Duration::from_secs(120) {
    break;
}

// 優化：延長到5分鐘
if last_message_time.elapsed() > Duration::from_secs(300) {
    break;
}
```

### 3. **優化配置參數**
```yaml
# 當前配置問題
connection:
  timeout_seconds: 45
  keepalive_interval_sec: 30

# 優化配置
connection:
  timeout_seconds: 60        # 增加超時時間
  keepalive_interval_sec: 25  # 更頻繁的心跳
  pong_tolerance: 3          # 容忍連續無響應次數
```

### 4. **自動重新平衡**
```rust
// 當前：手動提示
warn!("⚠️ 檢測到需要重新平衡，請手動調用 perform_rebalance()");

// 優化：自動執行
if let Err(e) = self.perform_auto_rebalance().await {
    error!("自動重新平衡失敗: {}", e);
}
```
